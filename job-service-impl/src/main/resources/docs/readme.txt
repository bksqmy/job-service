开发规范
1. Bug fix在主干上进行；新功能开发可以在主干或者分支上进行，新功能如果无并行项目则在主干开发，
   如果存在并行项目，则建立分支进行开发测试;
2. 生产环境部署使用tag下面给定版本号的代码；

主干trunk下面只允许保留release版本，不允许在主干上提交snapshot版本；
   新功能必须使用分支开发、测试：
   1.1 用主干代码建立分支;
   1.2 在分支上改为snapshot版本;
   1.3 在分支上完成开发、测试;
   1.4 将分支代码合并到主干并提交（只有在此时允许在主干上提交snapshot版本代码）;
       特别注意：在将snapshot代码提交到主干，尚未使用release插件完成发布期间，必须告知运维人员不得对该项目进行生产环境的发布、部署;
   1.5 针对主干使用maven release插件进行发布（release）;
   	   Maven release插件发布过程进行的操作如下:
   	   a). 将主干代码改成release版本并提交到svn;
   	   b). 在svn上建立tag并把release代码复制到tag下;
   	   c). 将release版本编译后相关的jar和pom.xml文件部署到maven仓库;
   	   d). 会将主干代码改成下一个snapshot版本并提交到svn;
